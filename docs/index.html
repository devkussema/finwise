FinWise Project Documentation

Welcome to the official documentation for the FinWise landing page. This guide serves as a central reference for developers and designers, detailing the project structure, visual style guide, reusable components, and implemented JavaScript functionalities.

The goal is to ensure consistency, facilitate maintenance, and accelerate the development of new features.

Project Structure

The project consists of a single HTML file that uses Tailwind CSS for styling and vanilla JavaScript for interactivity. External libraries like Feather Icons and AOS (Animate on Scroll) are imported via CDN.

/
└── index.html

Inside index.html, the structure is as follows:

  • <head>: Contains metadata, links to Google Fonts, and CDN imports (Tailwind, Feather Icons, AOS).
  • <style>: Defines CSS variables for theming (light/dark mode) and custom styles.
  • <body>: Houses the entire visible structure of the page, divided into <header>, <main>, and <footer>.
  • Modals: The contact and success modals are at the end of the <body>, initially hidden.
  • <script>: At the end of the <body>, contains the JavaScript for all interactive features.

JavaScript Functionality

JavaScript is responsible for all page interactivity. The code is wrapped in a DOMContentLoaded event listener to ensure the DOM is fully loaded before execution.

Modal Logic

Modal logic is handled by openModal and closeModal functions. Data attributes (data-modal-trigger and data-modal-close) are used to declaratively associate buttons with specific modals.

const openModal = (modal) => {
  if (!modal) return;
  modal.classList.remove('hidden');
  setTimeout(() => modal.classList.remove('opacity-0'), 10);
  // ...
};

const closeModal = (modal) => {
  if (!modal) return;
  modal.classList.add('opacity-0');
  // ...
  setTimeout(() => modal.classList.add('hidden'), 300);
};

// Add event listeners to all triggers and close buttons
document.querySelectorAll('[data-modal-trigger]').forEach(trigger => {
    // ...
});

Light/Dark Mode

The theme is controlled by adding or removing the dark class from the <html> element. The user's preference is saved in localStorage to persist between visits.

const setTheme = (theme) => {
    if (theme === 'dark') {
        htmlEl.classList.add('dark');
        localStorage.setItem('theme', 'dark');
    } else {
        htmlEl.classList.remove('dark');
        localStorage.setItem('theme', 'light');
    }
    feather.replace(); // Re-render icons for the new theme
};

// Check for saved theme or system preference on page load
const savedTheme = localStorage.getItem('theme') || (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
setTheme(savedTheme);
